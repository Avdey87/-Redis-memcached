# Задание 1. Кеширование
Приведите примеры проблем, которые может решить кеширование.
Проблема	Как решает кеширование
Высокая задержка ответа при частых запросах к тяжёлым ресурсам (SQL-запросы с JOIN-ами, вызовы внешних API, сложные расчёты)	Сохраняет результат первого запроса в оперативной памяти; последующие запросы отдаются за микросекунды, минуя медленные источники.
Чрезмерная нагрузка на основную БД (например, при пиковых нагрузках – распродажи, новогодние акции)	Кеш принимает на себя до 80‑90% read-запросов, снижая число соединений и CPU-циклов на ядре СУБД.
Повторное вычисление одних и тех же данных (например, рейтинги товаров, курсы валют, статистика)	Результат вычислений кешируется и переиспользуется, экономя ресурсы процессора.
Медленные внешние вызовы (запросы к платежным шлюзам, геосервисам, соцсетям)	Кеш хранит временные токены, координаты, результаты проверок – снижает количество внешних запросов и зависимость от чужих API.
Снижение пропускной способности при работе с сессиями пользователей	Кеш (например, Redis) хранит состояния сессий распределённо, позволяя балансировать нагрузку без привязки к одному серверу.
Пиковые всплески чтения (например, новость на главной странице)	Запись в кеш с коротким TTL позволяет выдержать тысячи одновременных просмотров без перегрузки базы.
Дорогие операции фильтрации/сортировки больших наборов данных	Кеширует предварительно агрегированные результаты (материализованные представления в памяти).
